Pruebas Funcionales Pruebas No Funcionales
Pruebas de unidad: se enfocan en probar cada unidad de código individualmente para verificar su funcionalidad. Pruebas de rendimiento: se enfocan en verificar la capacidad del software para manejar una gran cantidad de datos o usuarios simultáneamente.
Pruebas de integración: se enfocan en probar cómo interactúan las distintas unidades de código del software cuando se combinan. Pruebas de seguridad: se enfocan en verificar que el software está protegido contra ataques externos y que los datos están seguros.
Pruebas de sistema: se enfocan en probar el software completo y ver cómo se comporta en su conjunto. Pruebas de usabilidad: se enfocan en verificar que el software es fácil de usar y que los usuarios pueden realizar las tareas de manera eficiente.
Pruebas de aceptación: se enfocan en verificar que el software cumple con los requisitos y expectativas del cliente. Pruebas de compatibilidad: se enfocan en verificar que el software funciona correctamente en distintos sistemas operativos, navegadores y dispositivos.
Pruebas de escalabilidad: se enfocan en verificar que el software pueda manejar un aumento en la carga de trabajo sin afectar su rendimiento.
Pruebas de mantenibilidad: se enfocan en verificar que el software es fácil de mantener y actualizar.
